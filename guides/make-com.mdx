# Make.com Integration

Make.com (formerly Integromat) lets you connect Cold Navigator to thousands of apps with a visual drag-and-drop builder. Use webhooks to trigger automated scenarios whenever leads reply, connect, or move through your campaigns.

## Prerequisites

- A Cold Navigator account with an active API key
- A Make.com account (free tier works for testing)

<Steps>
  <Step title="Create a Custom Webhook in Make.com">
    Log in to Make.com and create a new scenario.

    1. Click the **+** button to add a module
    2. Search for **Webhooks** and select **Custom webhook**
    3. Click **Add** to create a new webhook
    4. Give it a name like `Cold Navigator Events`
    5. Click **Save**

    Make.com will generate a unique webhook URL. Copy this URL — you will need it in the next step.

    <Note>
      Do not close this tab. Make.com needs to stay in "listening" mode to receive the test event.
    </Note>
  </Step>

  <Step title="Create a Webhook in Cold Navigator">
    In Cold Navigator, navigate to **Settings > Webhooks** and click **Create Webhook**.

    1. **Name**: `Make.com Integration`
    2. **URL**: Paste the Make.com webhook URL you copied
    3. **Events**: Select the events you want to receive (e.g., `lead.replied`, `lead.connected`, `campaign.completed`)
    4. Click **Create**

  </Step>

  <Step title="Test the Connection">
    In Cold Navigator, find your webhook in the list and click the **Test** button. This sends a sample event to Make.com.

    Switch back to Make.com — you should see the test payload appear with a green checkmark. Make.com will automatically parse the JSON structure for you to use in subsequent modules.

    <Tip>
      If you do not see the test event, verify that your Make.com scenario is still in "listening" mode and that the webhook URL is correct.
    </Tip>
  </Step>

  <Step title="Build Your Scenario">
    Now that the webhook trigger is set up, add modules to process the events.

    **Recommended pattern:** Use a **Router** module after the webhook to handle different event types separately.

    ```
    Webhook Trigger
        |
      Router
       / \
      /   \
    lead.replied    lead.connected
        |                |
    Slack Message    Google Sheets Row
    ```

    1. Click the **+** after the webhook module
    2. Add a **Router** module
    3. For each route, set a filter based on the `event` field:
       - Route 1 filter: `event` equals `lead.replied`
       - Route 2 filter: `event` equals `lead.connected`
    4. Add action modules to each route
  </Step>

  <Step title="Example: Slack Notification + Google Sheets on Lead Reply">
    Here is a complete scenario that sends a Slack notification and logs the reply in Google Sheets when a lead replies.

    **Route 1 — Slack Notification:**

    1. Add a **Slack > Create a Message** module
    2. Select your Slack workspace and channel (e.g., `#sales-notifications`)
    3. Set the message text:

    ```
    New reply from {{data.lead.first_name}} {{data.lead.last_name}}
    Company: {{data.lead.company}}
    Campaign: {{data.campaign.name}}
    Message: {{data.message.text}}
    ```

    **Route 2 — Google Sheets:**

    1. Add a **Google Sheets > Add a Row** module
    2. Select your spreadsheet and sheet
    3. Map columns:
       - **Lead Name**: `{{data.lead.first_name}} {{data.lead.last_name}}`
       - **Company**: `{{data.lead.company}}`
       - **Reply**: `{{data.message.text}}`
       - **Date**: `{{created_at}}`
       - **Campaign**: `{{data.campaign.name}}`
  </Step>
</Steps>

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Make.com does not receive events | Verify the webhook URL is correct and the scenario is active (not paused) |
| Scenario runs but actions fail | Check module configurations and authentication for connected apps |
| Duplicate events | Enable Make.com's built-in deduplication or add an `id` filter |
| Events are delayed | Check Cold Navigator's webhook delivery log in **Settings > Webhooks > Deliveries** |

<Note>
  Make.com's free plan allows 1,000 operations per month. For high-volume outreach, consider upgrading to a paid plan.
</Note>
