# n8n Integration

n8n is an open-source workflow automation tool that you can self-host or use as a cloud service. It gives you full control over your data and integrations. This guide shows how to connect Cold Navigator webhooks to n8n workflows.

## Prerequisites

- A Cold Navigator account with an active API key
- An n8n instance (self-hosted or n8n Cloud)

<Steps>
  <Step title="Create a Webhook Node in n8n">
    Open your n8n editor and create a new workflow.

    1. Click **Add first step** (or the **+** button)
    2. Search for **Webhook** and select it
    3. Configure the node:
       - **HTTP Method**: `POST`
       - **Path**: Choose a path like `cold-navigator` (n8n will generate the full URL)
    4. Click **Listen for Test Event** to put n8n in listening mode

    n8n will show two URLs:
    - **Test URL**: For development (only works when the editor is open)
    - **Production URL**: For live use (works when the workflow is active)

    Copy the **Production URL** for the next step.

    <Note>
      Use the Test URL during setup to verify payloads. Switch to the Production URL before activating your workflow.
    </Note>
  </Step>

  <Step title="Create a Webhook in Cold Navigator">
    In Cold Navigator, go to **Settings > Webhooks** and click **Create Webhook**.

    1. **Name**: `n8n Workflow`
    2. **URL**: Paste the n8n webhook URL (use the Test URL first for testing)
    3. **Events**: Select the events you want (e.g., `lead.replied`, `lead.connected`, `campaign.completed`)
    4. **Secret**: Note the webhook secret — you will use it for signature verification
    5. Click **Create**

  </Step>

  <Step title="Test the Connection">
    In Cold Navigator, click the **Test** button next to your webhook. Switch back to n8n — you should see the test payload appear in the Webhook node.

    Click **Save test data** to store the sample payload. This allows n8n to show field mappings in downstream nodes.
  </Step>

  <Step title="(Optional) Enable HMAC Signature Verification">
    For production use, verify that webhook requests genuinely come from Cold Navigator by validating the `X-Webhook-Signature` header.

    Add a **Header Auth** credential in n8n:

    1. In the Webhook node, go to **Authentication** and select **Header Auth**
    2. Click **Create New Credential**
    3. Set:
       - **Header Name**: `X-Webhook-Signature`
       - **Header Value**: Your webhook secret from Cold Navigator

    Alternatively, use a **Code** node after the webhook to verify the HMAC-SHA256 signature programmatically:

    ```javascript
    const crypto = require('crypto');

    const secret = 'YOUR_WEBHOOK_SECRET';
    const signature = $input.first().headers['x-webhook-signature'];
    const timestamp = $input.first().headers['x-webhook-timestamp'];
    const body = JSON.stringify($input.first().json);

    // Cold Navigator signs: "${timestamp}.${body}"
    const signedContent = `${timestamp}.${body}`;
    const expectedSignature = crypto
      .createHmac('sha256', secret)
      .update(signedContent)
      .digest('hex');

    if (signature !== `sha256=${expectedSignature}`) {
      throw new Error('Invalid webhook signature');
    }

    return $input.all();
    ```

    <Tip>
      HMAC verification is strongly recommended for production deployments. It prevents attackers from sending fake events to your n8n webhook URL.
    </Tip>
  </Step>

  <Step title="Route Events with an IF Node">
    Use an **IF** node to route different event types to different actions.

    1. Add an **IF** node after the Webhook
    2. Set the condition: `{{ $json.event }}` equals `lead.replied`
    3. **True** branch: Handle reply events (e.g., update CRM)
    4. **False** branch: Add another IF or handle other events

    For multiple event types, use a **Switch** node instead:

    ```
    Webhook
       |
     Switch (event)
      /    |    \
    lead.  lead.  campaign.
    replied connected completed
      |      |        |
    CRM    Slack    Email
    Update  Alert   Summary
    ```
  </Step>

  <Step title="Example: CRM Update on Reply, Email Summary on Campaign Complete">
    **Branch 1 — Update CRM on lead.replied:**

    1. Add an **HTTP Request** node
    2. Configure it to update your CRM:
       - **Method**: `PATCH`
       - **URL**: `https://api.hubspot.com/crm/v3/objects/contacts/{{$json.data.lead.id}}`
       - **Body**: Map lead data and reply content

    **Branch 2 — Email summary on campaign.completed:**

    1. Add a **Send Email** node (Gmail, SMTP, or any email service)
    2. Configure:
       - **To**: `sales-team@yourcompany.com`
       - **Subject**: `Campaign "{{$json.data.campaign.name}}" completed`
       - **Body**:

    ```
    Campaign "{{$json.data.campaign.name}}" has completed.

    Results:
    - Total leads: {{$json.data.stats.total_leads}}
    - Replies: {{$json.data.stats.replies}}
    - Connections: {{$json.data.stats.connections}}

    View full results at https://app.coldnavigator.com/campaigns/{{$json.data.campaign.id}}
    ```
  </Step>
</Steps>

## Calling Cold Navigator API from n8n

You can use **HTTP Request** nodes to call Cold Navigator API endpoints. For example, to update a lead status:

```javascript
// HTTP Request node configuration
// Method: PATCH
// URL: https://app.coldnavigator.com/api/v1/public/leads/{{$json.data.lead.id}}/status

// Headers:
// x-api-key: YOUR_API_KEY
// Content-Type: application/json

// Body:
{
  "status": "qualified"
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| n8n does not receive events | Make sure the workflow is **active** (toggled on) and you are using the Production URL |
| Test URL works but Production does not | Activate the workflow — Production URLs only work when the workflow is active |
| Signature verification fails | Verify that the secret matches exactly and the body is not modified before hashing |
| Events arrive out of order | Add a **Sort** node or use timestamps to handle ordering in your workflow |

<Note>
  If you are self-hosting n8n, make sure your server is publicly accessible (or use a tunnel like ngrok for testing) so Cold Navigator can deliver webhook events.
</Note>
