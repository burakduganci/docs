# Event Types

Cold Navigator supports webhook event types organized into 6 categories. When creating or updating a webhook, specify event types using the dot-notation names listed below.

## Summary

| Category | Event Type | Description |
|----------|-----------|-------------|
| **Lead** | `lead.created` | A new lead was added to the workspace |
| | `lead.updated` | A lead's fields were modified |
| | `lead.status_changed` | A lead's status changed (e.g., new → contacted) |
| | `lead.replied` | A lead replied to a campaign message |
| | `lead.connected` | A lead accepted a connection request |
| **Campaign** | `campaign.started` | A campaign began executing |
| | `campaign.completed` | A campaign finished processing all enrolled leads |
| | `campaign.paused` | A campaign was paused (manually or by system) |
| | `campaign.error` | A campaign encountered an error *(reserved)* |
| **Message** | `message.sent` | An outbound message was delivered to a lead |
| | `message.received` | An inbound message was received from a contact |
| **Connection** | `connection.sent` | A connection request was sent to a lead |
| | `connection.accepted` | A lead accepted the connection request |
| **Workflow** | `workflow.step_completed` | A lead completed a step in the campaign workflow |
| | `workflow.lead_finished` | A lead finished the entire workflow |
| **Account** | `account.disconnected` | A LinkedIn account lost its session |
| | `account.rate_limited` | A LinkedIn account hit rate limits |

## Lead Events

Events triggered when leads are created, modified, or interact with your outreach.

### lead.created

Fired when a new lead is added to your workspace — through the UI, API, CSV import, or Sales Navigator import.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The created lead (see [Lead object](#lead-object)) |
| `source` | string | How the lead was created: `app`, `api`, `csv`, or `sales_navigator` |

<Accordion title="Example payload">
```json
{
  "id": "evt_m3k9x2p7_a1b2c3d4e5f6g7h8",
  "event": "lead.created",
  "created_at": "2026-03-06T14:23:17.482Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "lead": {
      "id": "f7e6d5c4-b3a2-1098-7654-321fedcba098",
      "first_name": "Sarah",
      "last_name": "Chen",
      "linkedin_url": "https://www.linkedin.com/in/sarachen",
      "company": "Acme Corp",
      "position": "VP of Engineering",
      "email": "sarah@acme.com",
      "status": "new"
    },
    "source": "app"
  }
}
```
</Accordion>

### lead.updated

Fired when a lead's profile fields are modified (name, company, position, email, etc.).

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The updated lead (see [Lead object](#lead-object)) |
| `updated_fields` | string[] | List of field names that changed |
| `changes` | object | Key-value map of changed fields with their new values |

<Accordion title="Example payload">
```json
{
  "id": "evt_m3k9x2p7_b2c3d4e5f6g7h8i9",
  "event": "lead.updated",
  "created_at": "2026-03-06T15:10:33.120Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "lead": {
      "id": "f7e6d5c4-b3a2-1098-7654-321fedcba098",
      "first_name": "Sarah",
      "last_name": "Chen",
      "linkedin_url": "https://www.linkedin.com/in/sarachen",
      "company": "Acme Corp",
      "position": "CTO",
      "email": "sarah@acme.com",

    },
    "updated_fields": ["position", "email"],
    "changes": {
      "position": "CTO",
      "email": "sarah@acme.com"
    }
  }
}
```
</Accordion>

### lead.status_changed

Fired when a lead's pipeline status changes. The payload includes both the previous and new status values.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The lead with updated status (see [Lead object](#lead-object)). Includes `previous_status`. |
| `old_status` | string | The previous status value |
| `new_status` | string | The new status value |

**Valid statuses:** `new`, `contacted`, `engaged`, `qualified`, `converted`, `unqualified`, `lost`, `replied`, `connected`, `do_not_contact`

<Accordion title="Example payload">
```json
{
  "id": "evt_m3k9x2p7_c3d4e5f6g7h8i9j0",
  "event": "lead.status_changed",
  "created_at": "2026-03-06T16:45:12.891Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "lead": {
      "id": "f7e6d5c4-b3a2-1098-7654-321fedcba098",
      "first_name": "Sarah",
      "last_name": "Chen",
      "linkedin_url": "https://www.linkedin.com/in/sarachen",
      "company": "Acme Corp",
      "position": "VP of Engineering",
      "email": null,
      "status": "qualified",
      "previous_status": "contacted"
    },
    "old_status": "contacted",
    "new_status": "qualified"
  }
}
```
</Accordion>

### lead.replied

| `campaign` | object | `{ id, name }` — the campaign the lead is enrolled in |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account that received the reply |
| `message` | object | `{ text }` — the reply message text (truncated to 500 chars) |

### lead.connected

Fired when a lead accepts your LinkedIn connection request (campaign workflow only).

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The lead who connected (see [Lead object](#lead-object)) |
| `campaign` | object | `{ id, name }` — the campaign |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account |
| `connected_at` | string | ISO 8601 timestamp of when the connection was accepted |

## Campaign Events

Events triggered by campaign lifecycle changes.

### campaign.started

Fired when a campaign transitions to the active state and begins processing leads.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `campaign_id` | string | UUID of the campaign |
| `campaign_name` | string | Name of the campaign |
| `total_leads` | number | Number of leads enrolled |

<Accordion title="Example payload">
```json
{
  "id": "evt_m3l2n4p8_a1b2c3d4e5f6g7h8",
  "event": "campaign.started",
  "created_at": "2026-03-06T10:00:00.000Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "campaign_id": "c9d8e7f6-5432-1abc-def0-987654321abc",
    "campaign_name": "Q1 Enterprise Outreach",
    "total_leads": 250
  }
}
```
</Accordion>

### campaign.completed

Fired when all enrolled leads in a campaign have finished their workflow.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `campaign_id` | string | UUID of the campaign |
| `campaign_name` | string | Name of the campaign |
| `stats` | object | `{ total, completed, failed }` — final campaign statistics |

<Accordion title="Example payload">
```json
{
  "id": "evt_m3l2n4p8_b2c3d4e5f6g7h8i9",
  "event": "campaign.completed",
  "created_at": "2026-03-06T18:45:02.119Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "campaign_id": "c9d8e7f6-5432-1abc-def0-987654321abc",
    "campaign_name": "Q1 Enterprise Outreach",
    "stats": {
      "total": 250,
      "completed": 243,
      "failed": 7
    }
  }
}
```
</Accordion>

### campaign.paused

Fired when a campaign is paused, either manually by a user or automatically by the system (e.g., due to account disconnection).

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `campaign_id` | string | UUID of the campaign |
| `campaign_name` | string | Name of the campaign |
| `reason` | string | Why the campaign was paused: `manual`, `account_disconnected`, `rate_limited` |

### campaign.error

*Reserved for future use.* Will fire when a campaign encounters a critical error.

## Message Events

Events triggered when messages are sent or received through LinkedIn.

### message.sent

Fired when an outbound message is delivered to a lead on LinkedIn. Triggers from three sources: campaign workflow, inbox UI, and legacy conversations.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The message recipient (see [Lead object](#lead-object)). May have `id: null` if the recipient is not a saved lead. |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account that sent the message |
| `campaign` | object \| null | `{ id, name }` — present only for campaign messages |
| `message` | object | `{ type, text, source }` — message details |

**`message` object:**

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Always `linkedin_message` |
| `text` | string \| null | Message text (truncated to 500 chars) |
| `source` | string | Where the message was sent from: `campaign` or `inbox` |

<Accordion title="Example payload (inbox)">
```json
{

  "event": "message.sent",
  "created_at": "2026-03-06T09:12:33.741Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "lead": {
      "id": "f7e6d5c4-b3a2-1098-7654-321fedcba098",
      "first_name": "Sarah",
      "last_name": "Chen",
      "linkedin_url": "https://www.linkedin.com/in/sarachen",
      "company": "Acme Corp",
      "position": "VP of Engineering",
      "email": null,
      "status": "contacted"
    },
    "linkedin_account": {
      "id": "d5e6f7a8-b9c0-1234-5678-9abcdef01234",
      "name": "John Sales"
    },
    "campaign": null,
    "message": {
      "type": "linkedin_message",
      "text": "Hi Sarah, I wanted to follow up on our conversation...",
      "source": "inbox"
    }
  }
}
```
</Accordion>

### message.received

Fired when an inbound message is received from any LinkedIn contact. This triggers for **all** incoming messages — not just leads enrolled in campaigns. Contacts without a linked lead record will have `lead.id: null` with name/URL extracted from the conversation.

Delivered via three paths for maximum reliability:
1. **SSE realtime** — instant, when the inbox SSE connection receives the message
2. **SSE conversation update** — backup, when LinkedIn delivers the message as a conversation update
3. **Delta sync** — periodic background sync catches any messages missed by SSE

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The message sender. `id` is `null` if the contact is not a saved lead. Name and URL are extracted from the conversation. |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account that received the message |
| `campaign` | object \| null | `{ id, name }` — present only if the conversation is linked to a campaign |
| `message` | object | `{ text }` — message content (truncated to 500 chars) |

<Accordion title="Example payload">
```json
{
  "id": "evt_m3m5o7q9_b2c3d4e5f6g7h8i9",
  "event": "message.received",
  "created_at": "2026-03-06T14:30:22.100Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "lead": {
      "id": "f7e6d5c4-b3a2-1098-7654-321fedcba098",
      "first_name": "Sarah",
      "last_name": "Chen",
      "linkedin_url": "https://www.linkedin.com/in/sarachen",
      "company": "Acme Corp",
      "position": "VP of Engineering",
      "email": null,
      "status": "replied"
    },
    "linkedin_account": {
      "id": "d5e6f7a8-b9c0-1234-5678-9abcdef01234",
      "name": "John Sales"
    },
    "campaign": {
      "id": "c9d8e7f6-5432-1abc-def0-987654321abc",
      "name": "Q1 Enterprise Outreach"
    },
    "message": {
      "text": "Sounds great, let's schedule a call for Tuesday."
    }
  }
}
```
</Accordion>

<Accordion title="Example payload (contact without lead record)">
```json
{
  "id": "evt_m3m5o7q9_c3d4e5f6g7h8i9j0",
  "event": "message.received",
  "created_at": "2026-03-06T15:45:10.333Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "lead": {
      "id": null,
      "first_name": "Alex",
      "last_name": "Johnson",
      "linkedin_url": "https://www.linkedin.com/in/alexjohnson",
      "company": null,
      "position": null,
      "email": null,
      "status": null
    },
    "linkedin_account": {
      "id": "d5e6f7a8-b9c0-1234-5678-9abcdef01234",
      "name": "John Sales"
    },
    "campaign": null,
    "message": {
      "text": "Hey, saw your post about cold outreach. Would love to chat."
    }
  }
}
```

<Info>
  **`message.received` vs `lead.replied`**: Use `message.received` to capture all incoming LinkedIn messages regardless of source. Use `lead.replied` to only capture replies from leads actively enrolled in campaigns.
</Info>

**`data` fields:**

| Field | Type | Description |

| `campaign` | object | `{ id, name }` — the campaign |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account |

## Workflow Events

Events triggered as leads progress through campaign workflow steps.

### workflow.step_completed

Fired each time a lead completes a step in the campaign workflow (e.g., connection request sent, message delivered, wait period finished).

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `lead` | object | The lead (see [Lead object](#lead-object)) |
| `campaign` | object | `{ id, name }` — the campaign |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account |
| `workflow_step` | object | `{ type, action_type }` — the completed step |

**`workflow_step.action_type` values:** `connection_request`, `send_message`, `send_inmail`, `like_post`, `follow`, `endorse_skill`, `view_profile`, `send_email`

### workflow.lead_finished

Fired when a lead has completed all steps in the workflow and exits the campaign.

**`data` fields:**

| Field | Type | Description |

| `campaign` | object | `{ id, name }` — the campaign |
| `linkedin_account` | object | `{ id, name }` — the LinkedIn account |
| `final_status` | string | The lead's final workflow status: `completed` |

## Account Events

Events triggered by LinkedIn account status changes. These are critical for maintaining healthy automation.

### account.disconnected

Fired when a LinkedIn account session becomes invalid. All campaigns using this account are automatically paused.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `linkedin_account_id` | string | UUID of the disconnected account |
| `account_name` | string | Display name of the account |
| `reason` | string | Why the account was disconnected (e.g., `SESSION_KILLED`, `SESSION_EXPIRED`) |

<Accordion title="Example payload">
```json
{
  "id": "evt_m3p4r6t8_f6g7h8i9j0k1l2m3",
  "event": "account.disconnected",
  "created_at": "2026-03-06T03:17:44.651Z",
  "api_version": "2026-03-01",
  "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "linkedin_account_id": "d5e6f7a8-b9c0-1234-5678-9abcdef01234",
    "account_name": "John Sales",
    "reason": "SESSION_KILLED"
  }
}
```
</Accordion>

### account.rate_limited

Fired when a LinkedIn account hits LinkedIn's usage limits. Cold Navigator automatically throttles the account.

**`data` fields:**

| Field | Type | Description |
|-------|------|-------------|
| `linkedin_account_id` | string | UUID of the rate-limited account |
| `account_name` | string | Display name of the account |
| `limit_type` | string | The action type that was rate-limited (e.g., `connection_request`, `send_message`) |

<Warning>
  Account events should trigger alerts in your system. A disconnected account means all associated campaigns will stop processing leads until the account is reconnected.
</Warning>

## Common Objects

### Lead object

The `lead` object is included in most webhook events and has a consistent structure:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string \| null | UUID of the lead. `null` for contacts not saved as leads (in `message.received`). |
| `first_name` | string \| null | Lead's first name |
| `last_name` | string \| null | Lead's last name |
| `linkedin_url` | string \| null | Full LinkedIn profile URL |
| `company` | string \| null | Current company |
| `position` | string \| null | Current job title |
| `email` | string \| null | Email address (if available) |
| `status` | string \| null | Current pipeline status |

### LinkedIn Account object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | UUID of the LinkedIn account |
| `name` | string \| null | Display name of the account |

### Campaign object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | UUID of the campaign |
| `name` | string \| null | Campaign name |
