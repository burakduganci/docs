# Webhooks

Webhooks let your application receive real-time notifications when events happen in Cold Navigator. Instead of polling the API to check for changes, Cold Navigator sends an HTTP POST request to your endpoint whenever a lead replies, a connection is accepted, a campaign completes, or any other tracked event occurs.

## How webhooks work

<Steps>
  <Step title="Create a webhook">
    Register a webhook endpoint through the [dashboard](https://app.coldnavigator.com/settings/webhooks) or the [API](/api-reference/webhooks/create-webhook). Provide a URL and select which events you want to receive.
  </Step>
  <Step title="Select events">
    Choose from 18 event types across leads, campaigns, messages, connections, workflows, and accounts. Subscribe only to the events you need.
  </Step>
  <Step title="Receive HTTP POST requests">
    When a subscribed event occurs, Cold Navigator sends a JSON payload to your endpoint via HTTP POST. Your server should respond with a `2xx` status code within 10 seconds.
  </Step>
  <Step title="Verify the signature">
    Each delivery includes an `X-Webhook-Signature` header. Use your webhook secret to [verify the signature](/webhooks/signature-verification) and confirm the request came from Cold Navigator.
  </Step>
</Steps>

## Webhook lifecycle

```
Event occurs in Cold Navigator
        |
        v
  Build JSON payload
        |
        v
  Sign with HMAC-SHA256
        |
        v
  POST to your endpoint
        |
        v
  2xx response? -----> Done
        |
       No
        v
  Retry (up to 5 attempts)
        |
        v
  All retries failed? --> Auto-disable webhook
```

<Note>
  Cold Navigator waits up to **10 seconds** for your endpoint to respond. If your processing takes longer, return a `202 Accepted` immediately and handle the event asynchronously.
</Note>

## Setting up a webhook

### Via the dashboard

1. Go to **Settings > Webhooks** in your Cold Navigator dashboard
2. Click **Create Webhook**
3. Enter your endpoint URL (must be HTTPS in production)
4. Select the events you want to subscribe to
5. Copy the signing secret and store it securely
6. Click **Save**

## Testing your webhook

After creating a webhook, click the **Test** button in **Settings > Webhooks**. This sends a sample payload with realistic test data (including a sample lead object) to your endpoint.

<Warning>
  The secret key is only shown once when the webhook is created. Store it immediately. If you lose it, use the **Rotate Secret** button in the webhook settings to generate a new one.
</Warning>

Cold Navigator webhooks work out of the box with popular automation platforms:

<Note>
  Any platform or service that can receive HTTP POST requests with JSON bodies will work with Cold Navigator webhooks. No special adapter or plugin is required.
</Note>

## Requirements for your endpoint

- **HTTPS** is required for production webhooks (HTTP is allowed for `localhost` during development)
- Must respond with a **2xx status code** within **10 seconds**
- Must accept **POST** requests with `Content-Type: application/json`
- Should be **idempotent** -- the same event may be delivered more than once in rare cases. Use the `id` field to deduplicate.
