# Quickstart

Follow these steps to make your first API call, create a lead, and set up a webhook to receive real-time events.

<Steps>
  <Step title="Create an API key">
    Log in to [Cold Navigator](https://app.coldnavigator.com) and navigate to **Settings > API Keys**. Click **Create API Key** and give it a descriptive name (e.g., "CRM Integration").

    Select the scopes you need. For this quickstart, select **read:leads** and **write:leads**.

    <Warning>
      Copy your API key immediately after creation. It will only be shown once. Store it securely — treat it like a password.
    </Warning>

    Your API key will look something like this:

    ```
    cn_live_a1b2c3d4e5f6...
    ```
  </Step>

  <Step title="Make your first API call">
    List your existing leads to verify the key works. Pass your API key in the `x-api-key` header.

    <CodeGroup>
      ```bash cURL
      curl -X GET https://app.coldnavigator.com/api/v1/public/leads \
        -H "x-api-key: YOUR_API_KEY"
      ```

      ```javascript JavaScript
      const response = await fetch("https://app.coldnavigator.com/api/v1/public/leads", {
        method: "GET",
        headers: {
          "x-api-key": "YOUR_API_KEY"
        }
      });

      const { data } = await response.json();
      console.log(data);
      ```
    </CodeGroup>

    You should receive a response like:

    ```json
    {
      "data": {
        "leads": [],
        "total": 0,
        "limit": 50,
        "offset": 0
      },
      "message": "Success"
    }
    ```

    <Note>
      If you get a `401` error, double-check that your API key is correct and active.
    </Note>
  </Step>

  <Step title="Get a lead group ID">
    Every lead must belong to a lead group. List your existing groups to get a `lead_group_id`:

    ```bash
    curl -X GET https://app.coldnavigator.com/api/v1/public/lead-groups \
      -H "x-api-key: YOUR_API_KEY"
    ```

    Copy the `id` of the group you want to add leads to. If you don't have one yet, create one:

    ```bash
    curl -X POST https://app.coldnavigator.com/api/v1/public/lead-groups \
      -H "x-api-key: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"name": "My First Group"}'
    ```
  </Step>

  <Step title="Create a lead">
    Add a new lead by sending a `POST` request with the lead details. `first_name`, `last_name`, `linkedin_url`, and `lead_group_id` are required.

    <CodeGroup>
      ```bash cURL
      curl -X POST https://app.coldnavigator.com/api/v1/public/leads \
        -H "x-api-key: YOUR_API_KEY" \
        -H "Content-Type: application/json" \
        -d '{
          "first_name": "Jane",
          "last_name": "Smith",
          "linkedin_url": "https://www.linkedin.com/in/janesmith",
          "lead_group_id": "YOUR_LEAD_GROUP_UUID",
          "company": "Acme Corp",
          "position": "VP of Sales"
        }'
      ```

      ```javascript JavaScript
      const response = await fetch("https://app.coldnavigator.com/api/v1/public/leads", {
        method: "POST",
        headers: {
          "x-api-key": "YOUR_API_KEY",
          "Content-Type": "application/json"
        },
        body: JSON.stringify({
          first_name: "Jane",
          last_name: "Smith",
          linkedin_url: "https://www.linkedin.com/in/janesmith",
          lead_group_id: "YOUR_LEAD_GROUP_UUID",
          company: "Acme Corp",
          position: "VP of Sales"
        })
      });

      const { data } = await response.json();
      console.log("Created lead:", data.id);
      ```
    </CodeGroup>

    A successful response returns the created lead with a `201` status:

    ```json
    {
      "data": {
        "id": "f7e6d5c4-b3a2-1098-7654-321fedcba098",
        "first_name": "Jane",
        "last_name": "Smith",
        "linkedin_url": "https://www.linkedin.com/in/janesmith",
        "headline": null,
        "email": null,
        "phone": null,
        "company": "Acme Corp",
        "position": "VP of Sales",
        "location": null,
        "status": "new",
        "custom_fields": null,
        "created_at": "2026-03-06T10:30:00Z",
        "updated_at": "2026-03-06T10:30:00Z"
      },
      "message": "Lead created"
    }
    ```

    <Note>
      If a lead with the same `linkedin_url` already exists in your workspace, the API returns `200` with message "Lead already exists, added to group" instead of creating a duplicate.
    </Note>
  </Step>

  <Step title="Set up a webhook">
    Webhooks let you receive real-time notifications when events happen — like a lead replying to a message or accepting a connection request.

    Navigate to **Settings > Webhooks** in the Cold Navigator dashboard and click **Create Webhook**.

    1. **Name**: Give it a descriptive name (e.g., "CRM Sync")
    2. **URL**: Enter your endpoint URL (e.g., `https://your-app.com/webhooks/cold-navigator`)
    3. **Events**: Select the events you want to receive (e.g., `lead.replied`, `lead.connected`, `lead.status_changed`)
    4. Click **Create**

    <Warning>
      Save the **secret key** shown after creation. You will need it to verify that incoming webhook payloads are genuinely from Cold Navigator. See [Signature Verification](/webhooks/signature-verification) for details.
    </Warning>

    Once created, click the **Test** button next to your webhook. If you select a specific event type to test (e.g., `lead.replied`), the test payload will match that event's schema with sample data. If you test without selecting an event type, your endpoint will receive a generic test payload:

    ```json
    {
      "id": "evt_test_abc123",
      "event": "webhook.test",
      "created_at": "2026-03-06T10:36:00Z",
      "api_version": "2026-03-01",
      "data": {
        "message": "This is a test webhook delivery from Cold Navigator",
        "webhook_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "timestamp": "2026-03-06T10:36:00Z"
      }
    }
    ```

    For example, testing with `lead.replied` sends:

    ```json
    {
      "id": "evt_test_abc123",
      "event": "lead.replied",
      "created_at": "2026-03-06T10:36:00Z",
      "api_version": "2026-03-01",
      "data": {
        "lead": {
          "id": "00000000-0000-0000-0000-000000000001",
          "first_name": "Jane",
          "last_name": "Smith",
          "company": "Acme Corp",
          "position": "VP of Sales",
          "email": "jane@acme.com",
          "status": "new"
        },
        "campaign": {
          "id": "00000000-0000-0000-0000-000000000002",
          "name": "Q1 Outreach"
        },
        "linkedin_account": {
          "id": "00000000-0000-0000-0000-000000000003",
          "name": "John Doe"
        },
        "message": {
          "text": "Hi, thanks for reaching out! I would love to learn more."
        }
      }
    }
    ```

    <Note>
      If your endpoint does not respond with a `2xx` status code within 10 seconds, the delivery will be marked as failed. Cold Navigator retries failed deliveries up to 5 times with exponential backoff.
    </Note>
  </Step>
</Steps>
